FAQ

A branch = your own parallel universe of code. You branch off, experiment, and merge back when ready.

main        ●───●───●───────────────●  (stable)
                 \                 /
feature          ●───●───●───●──●   (your work)
                 ↑               ↑
            git branch      git merge
           feature          feature → main
git branch feature

Create branch

git checkout feature

Switch to it

git merge feature

Merge back to main

Real-world scenarios — why branching saves your life

🚀 Scenario 1 — New feature without breaking production

Your app is live. Boss says "add dark mode". You create git branch dark-mode, build it safely. Meanwhile, main stays untouched — users never see half-done work.

🔥 Scenario 2 — Emergency hotfix while feature is in progress

You're halfway through building a payment page on feature/payments. Suddenly — critical bug in production! You switch to git checkout main, create hotfix/login-crash, fix it, merge, deploy. Then go right back to your payments branch. No code mixed up.

👥 Scenario 3 — Two devs, same project, zero conflicts

Dev A works on feature/navbar, Dev B works on feature/footer. Both branch from main. They never step on each other's code. When done, both merge back — clean and independent.

🧪 Scenario 4 — "Let me try something crazy"

Want to rewrite the entire auth system? Create experiment/new-auth. If it works — merge it. If it's a disaster — just delete the branch. Main is still perfect. Zero risk.

Offline work?❌ No✅ Yes

Think of it as a save point in a video game — you can always go back to it.

  Working Dir    Staging Area     Repository
  ┌────────┐     ┌────────┐      ┌────────┐
  │ edit   │────►│ staged │─────►│ saved! │
  │ files  │     │ files  │      │ forever│
  └────────┘     └────────┘      └────────┘
    git add .      git commit -m "msg"

It's like raising your hand and saying "Hey team, I'm done — please review before I merge."

  You (feature branch)
    │
    ├── code, code, code...
    ├── git push origin feature
    │
    ▼
  Open Pull Request on GitHub
    │
    ├── 👀 Team reviews your code
    ├── 💬 Comments & suggestions
    ├── ✅ Approved!
    │
    ▼
  Merge → main branch updated

  ☀️ Morning
  git pull origin main          ← get latest changes
  git checkout -b my-task       ← create your branch

  💻 During the day
  git add .                      ← stage your work
  git commit -m "built feature" ← save locally

  🌙 End of day
  git push origin my-task       ← push to GitHub
  Open a Pull Request           ← request review
